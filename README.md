# Chzzk-VTS Bridge

**빙하유 사랑해**

치지직(CHZZK) 후원/채팅에 반응하여 VTuber Studio 액션을 자동 실행하는 브릿지 앱입니다.
웹 대시보드에서 모든 설정을 GUI로 관리할 수 있습니다.

---

## 주요 기능

- **후원 반응**: 치지직 후원(치즈/영상)이 들어오면 VTuber Studio 반응 자동 실행
- **채팅 반응**: 특정 채팅 키워드에 반응하여 VTuber Studio 반응 자동 실행
- **웹 대시보드**: `http://localhost:3000`에서 설정/규칙 관리
- **초기 설정 위자드**: 처음 사용할 때 단계별 안내
- **테스트 기능**: 실제 후원/채팅 없이 가짜 이벤트로 규칙 테스트
- **VTS 액션 5종**: 핫키 실행, 표정 변경, 아이템 표시, 아이템 던지기, 얼굴 색 변경

## 설치

### 방법 1: exe 파일로 바로 실행 (추천)

Node.js 설치가 필요 없습니다.

1. [Releases](https://github.com/hj529ho/Chzzk-VTS-Bridge/releases) 페이지에서 최신 zip 파일 다운로드
2. 원하는 폴더에 압축 해제
3. `chzzk-vts-bridge.exe` 실행
4. 브라우저에서 `http://localhost:3000` 접속

> 설정 파일(`config.json`, `settings.json` 등)은 exe와 같은 폴더에 자동 생성됩니다.

### 방법 2: 소스코드에서 실행 (개발자용)

#### 1. 저장소 클론 및 의존성 설치

```bash
git clone <repository-url>
cd chzzk-vts-bridge
npm install
```

#### 2. 서버 실행

```bash
npm run dev
```

브라우저에서 `http://localhost:3000`에 접속하면 초기 설정 위자드가 나옵니다.

**브라우저에서 초기 설정을 마치면 브라우저는 종료하셔도 됩니다.(터미널(서버)는 종료하시면 안됩니다.)**

## 초기 설정 (웹 대시보드 위자드)

서버를 처음 실행하면 3단계 위자드가 표시됩니다.

### Step 1: 치지직 개발자 등록

1. [치지직 개발자센터](https://developers.chzzk.naver.com)에 접속하여 로그인
2. "새 애플리케이션 등록" 클릭
3. 앱 이름: 아무거나 (예: "내 후원 연동")
4. **로그인 리다이렉트 URL**에 아래 주소를 입력:
   ```
   http://localhost:3000/auth/callback
   ```
5. 등록 후 발급되는 **Client ID**와 **Client Secret**을 대시보드에 입력
6. **채널 ID** 입력 (치지직 내 채널 페이지 URL의 마지막 부분)

### Step 2: 치지직 계정 연결

1. "치지직 계정 연결하기" 버튼 클릭
2. 치지직 로그인 페이지에서 로그인
3. 완료되면 자동으로 창이 닫힘

### Step 3: VTuber Studio 연결

1. VTuber Studio 실행
2. 왼쪽 메뉴 > 톱니바퀴(설정) 클릭
3. **"API (Plugin) 사용"**을 **켜기**로 변경
4. "Chzzk Donation Bridge" 플러그인 승인 팝업에서 **허용** 클릭

설정이 완료되면 대시보드 메인 화면으로 이동합니다.

## 사용법

### 규칙 만들기

대시보드에서 **"+ 새 규칙 만들기"** 버튼을 클릭합니다.

#### 트리거 유형

| 유형 | 설명 |
|------|------|
| **후원이 들어오면** | 치지직 후원(치즈/영상)에 반응 |
| **채팅이 올라오면** | 치지직 채팅 메시지에 반응 |
| **후원 + 채팅 모두** | 둘 다에 반응 |

#### 후원 조건 (후원 트리거일 때)

| 조건 | 설명 |
|------|------|
| 최소/최대 금액 | 후원 금액 범위 필터 (비우면 제한 없음) |
| 후원 종류 | 채팅 후원(치즈), 영상 후원, 또는 전부 |

#### 채팅 조건 (채팅 트리거일 때)

| 조건 | 설명 |
|------|------|
| 특정 닉네임만 | 쉼표로 구분 (예: `팬1, 팬2`). 비우면 모든 사람 |

#### 공통 조건

| 조건 | 설명 |
|------|------|
| 특정 단어 포함 | 쉼표로 구분 (예: `사랑, 좋아`). 비우면 모든 메시지 |
| 같은 반응 대기 시간 | 같은 규칙이 연속 실행되지 않게 쉬는 시간 (초) |

#### 액션 종류

| 액션 | 설명 |
|------|------|
| **핫키 실행** | VTuber Studio에 설정된 핫키를 실행 |
| **표정 변경** | 모델의 표정(Expression)을 변경. 유지 시간 설정 가능 |
| **아이템 표시** | 아이템 이미지를 화면에 표시. 위치/크기/유지 시간 설정 가능 |
| **아이템 던지기** | 아이템을 화면 밖에서 모델에게 던지기. 속도/흔들림 강도/유지 시간 설정 가능 |
| **얼굴 색 변경** | 모델 얼굴에 색상 틴트 적용. 유지 시간 설정 가능 |

하나의 규칙에 여러 액션을 추가할 수 있고, 각 액션에 실행 지연(초)을 설정하여 순서대로 연출할 수 있습니다.

### 테스트

대시보드 하단의 **"테스트 보내기"** 섹션에서 가짜 후원/채팅으로 규칙 동작을 확인할 수 있습니다.

- **후원 테스트**: 닉네임, 금액, 메시지를 입력하고 "테스트 후원 보내기"
- **채팅 테스트**: 닉네임, 채팅 내용을 입력하고 "테스트 채팅 보내기"

규칙 카드의 **"테스트"** 버튼으로 개별 규칙의 액션만 직접 실행할 수도 있습니다.

## 규칙 예시

### 1000원 이상 후원 시 환호 표정

- 트리거 유형: **후원이 들어오면**
- 최소 금액: `1000`
- 액션: **표정 변경** > 원하는 표정 선택, 유지 시간 `3`초

### "사랑" 채팅에 하트 이모지 핫키

- 트리거 유형: **채팅이 올라오면**
- 특정 단어 포함: `사랑, 좋아, 최고`
- 액션: **핫키 실행** > 하트 핫키 선택
- 같은 반응 대기 시간: `10`초 (도배 방지)

### 10000원 이상 대형 후원 풀 연출

- 트리거 유형: **후원이 들어오면**
- 최소 금액: `10000`
- 액션 1: **표정 변경** > 놀람 표정, 유지 시간 `5`초
- 액션 2: **아이템 표시** > 축하 이미지, 유지 시간 `5`초, 실행 지연 `0.5`초
- 액션 3: **얼굴 색 변경** > 분홍색, 유지 시간 `3`초, 실행 지연 `1`초

```

├── index.ts                 # 진입점 (서버 + VTS + 치지직 시작)

│   └── session.ts           # 치지직 Session WebSocket (후원/채팅 수신)

│   ├── client.ts            # VTuber Studio API 연결
│   ├── discovery.ts         # VTS 핫키/표정/아이템 목록 조회
│   └── actions.ts           # VTS 액션 실행 (핫키/표정/아이템/틴트)
├── bridge/
│   ├── engine.ts            # 이벤트 → 규칙 매칭 → 액션 실행 오케스트레이션
│   ├── rule-matcher.ts      # 후원/채팅 규칙 매칭 로직
│   └── action-executor.ts   # 액션 실행 + 쿨다운 관리
├── config/
│   ├── schema.ts            # Zod 스키마 (규칙/액션/설정 구조 정의)
│   └── loader.ts            # config.json 로드/저장/감시
├── server/
│   ├── app.ts               # Fastify 서버 설정
│   ├── routes/
│   │   ├── api.ts           # REST API (규칙 CRUD, 설정, VTS 조회, 테스트)
│   │   ├── auth.ts          # 치지직 OAuth 콜백 라우트
│   │   └── websocket.ts     # 대시보드 WebSocket 실시간 이벤트
│   └── static/
│       ├── index.html       # 대시보드 HTML
│       ├── style.css         # 대시보드 스타일
│       └── app.js           # 대시보드 프론트엔드 로직
└── utils/
    ├── event-bus.ts          # 타입 안전 이벤트 버스
    └── logger.ts             # Pino 로거
```

## 문제 해결

### 포트가 이미 사용 중 (EADDRINUSE)

```bash
npx kill-port 3000
npm run dev
```

### 치지직 연결 안 됨

- 방송 중인지 확인하세요 (채팅 이벤트는 방송 중에만 수신됩니다)
- 대시보드 > 설정에서 Client ID/Secret이 올바른지 확인하세요
- 치지직 개발자센터에서 리다이렉트 URL이 `http://localhost:3000/auth/callback`인지 확인하세요

### VTuber Studio 연결 안 됨

- VTuber Studio가 실행 중인지 확인하세요
- 설정 > "API (Plugin) 사용"이 켜져 있는지 확인하세요
- 플러그인 승인 팝업이 뜨면 **허용**을 눌러주세요

### 핫키/표정 목록이 비어있음

- VTuber Studio가 먼저 연결되어야 합니다
- 모델을 변경하면 자동으로 목록이 갱신됩니다
- VTuber Studio에서 핫키/표정을 먼저 등록해주세요

### 터미널에 한글이 깨짐

`npm run dev`로 실행하면 자동으로 UTF-8(chcp 65001)이 설정됩니다. 수동 실행 시:

```bash
chcp 65001
npx tsx src/index.ts
```

## 업데이트 내역

### v0.2.0 — 아이템 던지기 & 다중 던지기

#### 아이템 던지기 액션
- 아이템을 화면 밖에서 모델 머리 방향으로 던지는 새 액션 추가
- 날아오는 속도, 아이템 크기, 흔들림 강도, 머리 높이 오프셋 설정 가능
- 맞은 후 거의 수직으로 튕겨올라갔다가 회전하며 포물선 낙하
- 모델 크기에 비례하여 아이템 크기/날아오는 거리/바운스 자동 스케일링
- 충돌 시 모델 흔들림 후 원위치 자동 복귀

#### 후원 금액 비례 다중 던지기
- "단가당 개수" 설정: 예) 1000원 설정 시 만원 후원 → 10개 동시 던지기
- "최대 개수" 설정으로 상한 제한 (기본 30개)
- 동시 던지기 시 원위치 자동 관리 (첫 throw가 저장, 마지막 throw가 복귀)
